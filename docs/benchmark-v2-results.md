### 지표 3: 검색 품질 (Search Quality)

|------|-----|-------|------|

| Q7: karpathy llm wiki | 100 | 90 | 60 |
| Q8: RAG pitfalls | 40 | 90 | 50 |

- **Q8**: cairn이 karpathy-llm-wiki-pattern entry에서 "RAG 대 wiki" 핵심 개념 정확 적중. owl은 weak hit.

owl은 Q1/Q3/Q7에서 강점, cairn은 Q8처럼 의미론적 반대 개념 검색에서 강점.

## 5. 서사 분석

### cairn: LLM-first brain의 최적 형태

cairn은 처음부터 LLM이 읽고 쓰는 것을 전제로 설계된 view다.  
INDEX.md 하나로 세션 시작 시 전체 뇌의 목차를 제공하고, atomic entry 포맷은 parallel agent가 동시에 쓸 수 있도록 최적화되어 있다. raw 1개 추가 시 `cairn index` 한 번으로 INDEX.md가 1초 내 갱신된다.

주요 약점은 두 가지다. 첫째, 키워드 기반 TF-IDF는 "loop"처럼 다의어 키워드에서 의미 충돌을 일으킨다(Q3 실패). 둘째, strict schema(400 토큰 제한 + Why it matters 필수)는 인체공학 점수를 낮춘다. 그러나 이 엄격함은 장기적으로 entry 품질과 일관성을 보장하는 트레이드오프다.

### owl: 성숙한 hybrid 시스템, 검색 품질에서 우위

owl은 v1 벤치마크부터 6주간 운영하며 축적된 인프라가 있다. YAML frontmatter와 자유 본문의 조합은 내용 밀도가 높고, TF-IDF 스코어링의 안정성이 일관성 지표에서 가장 높은 점수(90점)를 만들었다.

결정적 약점은 sequential compile이다. 쓰기 처리량(46.1점)과 Update 전파 비용(55점)이 cairn 대비 크게 뒤처진다. 또한 Startup 비용(50점) — CLAUDE.md는 존재하지만 전체 vault의 목차 역할을 하지 않는다.

### wiki: 철학은 올바르나 도구가 미완성

wiki는 Karpathy LLM wiki spec을 가장 순수하게 구현한 control group이다. plain markdown + bold 헤더의 간결함(인체공학 90점)과 raw grep의 압도적 속도(100점)는 장점이나, 실질적 사용 시나리오에서 치명적 약점을 드러낸다.

진입점 없음(Startup 0점), 검색 품질 최하위(32.5점), 일관성 최하위(30점). wiki는 "도구 없이 가장 간단하게 만든 Karpathy wiki"지만, 바로 그 단순함이 LLM 세션 컨텍스트와 통합하기 어렵게 만든다. grep은 빠르지만 랭킹이 없어 결과를 LLM이 해석해야 하는 추가 부담이 발생한다.

## 9. 한계 및 주의사항

- **관찰 쿼리 수 제한**: 검색 품질은 4개 쿼리만 실측됨. 10개 전체 쿼리셋으로 확장 시 결과 달라질 수 있음.
- **owl throughput 미측정**: owl의 쓰기 처리량은 실측이 아닌 추정값(~7 files/min)으로 신뢰도 낮음. 실측 시 점수 변동 가능.
- **wiki 진입점 부재**: wiki startup 0점은 구조적 결함이나, INDEX.md를 추가하면 개선 가능.
- **cairn Q3 실패**: "filing loop" 쿼리에서 keyword collision은 시맨틱 임베딩 도입으로 해결 가능한 기술 문제임.
- **가중치 사용자 지정**: 검색 품질 가중치를 20% → 25%로 올리면 owl이 2위로 순위 변동 없이 gap이 줄어들 수 있음.
- **벤치마크 설계 시점**: 본 결과는 2026-04-12 기준 스냅샷이며, 향후 view 업데이트로 수치 달라질 수 있음.

## 결론

**결론: 메인 뷰 = cairn, 이유 — 가중 종합 점수 77.33점으로 owl(66.81점), wiki(66.13점) 대비 10점 이상 우위. cairn은 쓰기 처리량(1위, 100점), Startup 비용(1위, 80점), Update 전파 비용(1위, 95점) 세 지표에서 독보적이며, 이 지표들은 LLM-first knowledge system의 핵심 운영 요소다. 검색 품질에서 owl(78.8점)에 소폭 뒤지나(cairn 70.0점), 해당 gap(−1.68 weighted points)은 나머지 지표들의 합산 우위(+12.20)로 상쇄된다. cairn을 메인 뷰로, owl을 인간 browsing 보조 뷰로, wiki를 철학 control group으로 운영하는 3-tier 체계를 권고한다.**
