## 새 볼트 구조 (v3.3 확정)

```
~/omb/
  input/            raw inputs (immutable)
  brain/            ← 쓰는 곳 (omb 가 답함)
    live/           실제 편집/축적되는 지식 (Karpathy LLM Wiki)
    readonly/       읽기 전용 제품 번들 (이전 capsule)
  bench/            ← 측정만 하는 곳 (응답 비사용)
    akasha/         이전 설계 백업 (도전자 #1)
    sandbox/        신규 볼트 슬롯 (도전자 #2 자리)
```

의미 축: **사용 여부**. 3 단계 위계:
- 최상위: `input` (원료) · `brain` (쓰는 곳) · `bench` (측정만)
- `brain` 내부: `live` (편집) · `readonly` (읽기 전용)
- `bench` 내부: 도전자들

## 외부 인터페이스 규약

사용자는 `omb` 명령만 쓴다. 내부 구현 용어(`wiki`, `akasha`, `owl`, `cairn`, `capsule`) 는 **출력·help 에 노출 금지**. 폴더 이름 (`brain/live`, `brain/readonly`, `bench`) 은 사용자가 같이 정했으므로 OK.

- `omb status / search / ingest` 의 기본 대상은 `brain/live`
- `search` 는 query 매칭 시 `brain/readonly` 의 해당 제품번들 자동 첨부
- `bench/` 접근은 `omb akasha <sub>` 명시 호출만. 결과는 로그 전용, 응답에 반영 안 됨
- 교체 조건: `bench/` 의 무언가가 `brain/live` 를 뛰어넘는 증거 제시 시 재평가

## 재설계 원칙 (짧게)

1. **Karpathy 원안 그대로.** 내가 "개선" 하지 않는다.
2. **drift 감지를 설계에 박는다.** 이전 설계가 이걸 놓쳤다.
3. **CLI 는 얇게.** omb = wiki 용 얇은 진입점 + capsule 자동 첨부. 그 외 기능은 기본 빼고, 필요 시 명시 호출.
4. **문서는 한 곳.** docs/ 하위에 여러 계약 문서를 쌓지 않는다. README + CHANGELOG + 이 REZERO + 볼트별 AGENTS.md. 끝.
5. **LLM 이 ingest 한 기록은 매번 log.md 에 남긴다.** 드리프트 추적의 근거.

## 기록 체계 (drift 대응)

- **source → wiki**: source 변경 시 wiki 해당 페이지 업데이트 + `log.md` append
- **source → CLAUDE.md / 매핑**: deprecation 같은 메타 변경은 **drift audit 스크립트**가 주 1회 대조. 차이 발견 시 경고
- **wiki ↔ akasha 동기 금지**: 두 진실 공존 = drift 원인. akasha 는 읽기 전용 백업
