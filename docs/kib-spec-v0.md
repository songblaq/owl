# KIB 명세 v0

작성일: 2026-04-03
상태: 설계 초안

## 1. 정의

KIB(Karpathy Info Base)는 owl 안에서 raw 자료와 compiled 지식 산출물을 보관하고 열람하게 하는 정보 기반 계층이다.

핵심은 다음 두 가지다.

- 원본 자료를 파일 단위로 보존한다.
- LLM이 원본을 읽고 만든 구조화 지식을 함께 보관한다.

즉 KIB는 단순 저장소가 아니라, **원본과 편찬 결과를 함께 유지하는 정보 기반**이다.

---

## 2. KIB의 데이터 유형

### Raw 데이터
예:
- 기사 원문
- 논문 PDF
- 클립된 markdown
- 저장소 README/문서 스냅샷
- 이미지
- 대화 로그
- 링크 모음
- 아이디어 메모

### Compiled 데이터
예:
- 위키 문서
- 주제 요약
- 개념 정리
- 인덱스 문서
- 비교 문서
- 보고서
- 슬라이드 초안

---

## 3. KIB의 기본 기능

### 3.1 Ingest
- 자료를 inbox/raw로 수집한다.
- 가능한 원본 그대로 둔다.
- 최소한의 출처 정보와 수집 맥락을 유지한다.

### 3.2 Preserve
- 원본 자료는 지식 편찬 후에도 남겨둔다.
- compiled 결과가 생겨도 raw는 삭제하지 않는다.

### 3.3 Compile
- LLM이 raw 자료를 읽고 compiled 문서를 만든다.
- 단순 요약뿐 아니라 분류, 연결, index, article generation 을 포함한다.

### 3.4 Browse
- 사람과 LLM이 모두 읽을 수 있게 한다.
- markdown 중심 구조를 기본으로 한다.

### 3.5 Re-file
- Q&A, 리서치, 보고서 결과를 다시 KIB로 넣는다.
- 결과물 역시 지식 자산으로 취급한다.

---

## 4. KIB 디렉토리와 역할

### `raw/`
원본 자료 보관

### `compiled/`
LLM이 편찬한 지식 문서 보관

### `research/`
조사 중간 산출물과 임시 정리물 보관

### `inbox/`
아직 분류되지 않은 수집 대기 자료

---

## 5. KIB의 설계 원칙

- 원본 우선 보존
- markdown 우선 열람
- LLM-maintained knowledge 지향
- 인간 친화적이면서도 LLM 친화적인 파일 구조 지향
- source와 compiled의 경계 유지

---

## 6. KIB가 하지 않는 것

- 복잡한 메타 차원 해석의 주체가 되지 않는다.
- 모든 판단 로직을 내장하려 하지 않는다.
- profile별 의미 해석은 MDV에 넘긴다.

즉 KIB는 실제 지식 파일의 기반이고, 메타 해석 엔진은 아니다.
