# Small-Scale No-RAG Policy v0

작성일: 2026-04-03
상태: 보완 초안

## 1. 목적

Karpathy 원문에서 말한 "small scale 에서는 fancy RAG 없이도 잘 작동한다"는 감각을 owl 초기 운영 정책으로 정리한다.

---

## 2. 기본 원칙

- 초기 단계에서는 복잡한 RAG 인프라를 전제하지 않는다.
- 잘 유지된 summary, index, concept 문서만으로도 충분히 질의응답 가능한 범위를 우선 활용한다.
- raw와 compiled 구조가 잘 정리되어 있으면 small scale 에서는 LLM이 중요한 문서를 비교적 쉽게 읽을 수 있다고 본다.

---

## 3. 적용 범위

다음과 같은 경우에 우선 적용한다.
- 문서 수가 아직 소규모일 때
- 주제 범위가 비교적 좁을 때
- summary와 index가 충분히 유지될 때
- ultra-low latency 검색이 필수는 아닐 때

---

## 4. 전환 기준

다음 경우에는 더 고도화된 검색/인덱싱/RAG를 검토할 수 있다.
- 문서량이 급격히 증가할 때
- 주제가 매우 넓고 교차 참조가 복잡할 때
- 응답 지연과 탐색 비용이 크게 증가할 때
- 반복 질의가 단순 파일 탐색만으로 어려워질 때

---

## 5. 운영 효과

이 정책은 초기 설계를 과도하게 무겁게 만들지 않고, KIB와 compiled wiki의 품질을 먼저 높이게 한다.
