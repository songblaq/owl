# Doc First Template v0

작성일: 2026-04-04
상태: 템플릿

## 사용 목적

중요한 판단, 설계, 작업, 결과를 문서 객체로 남길 때 사용하는 기본 템플릿이다.

---

# 제목

작성일: YYYY-MM-DD
상태: draft | active | done | archived
유형: plan | spec | task | report | concept | summary

## 목적

이 문서가 해결하려는 문제와 기대 결과를 적는다.

## 배경

왜 이 문서가 필요한지, 어떤 맥락에서 나왔는지 적는다.

## 범위

이번 문서가 다루는 범위와 다루지 않는 범위를 적는다.

## 입력

참고한 자료, 선행 문서, 관련 대화, 원본 데이터 경로를 적는다.

## 출력

기대 산출물이나 완료 시 남아야 할 파일/결과를 적는다.

## 제약

시간, 도구, 정책, 안전, 권한 같은 제약을 적는다.

## 실행 계획

1. 단계 1
2. 단계 2
3. 단계 3

## 검증 기준

완료 여부를 어떤 기준으로 확인할지 적는다.

## 결과

실제 수행 후 무엇이 만들어졌고 무엇이 바뀌었는지 적는다.

## 후속 조치

다음에 이어서 해야 할 일, 승격 여부, 보완점을 적는다.

## 관련 문서

연결해야 할 설계/작업/결과/지식 문서를 적는다.

---

## 빠른 사용 규칙

- 실행 전에는 `목적`, `범위`, `출력`, `검증 기준`을 우선 채운다.
- 실행 후에는 `결과`, `후속 조치`를 반드시 채운다.
- 재사용 가치가 생기면 `concept`, `index`, `report`로 승격한다.
