# Karpathy — LLM Knowledge Bases (한글 정리 보관)

출처: Andrej Karpathy 공개 글
보관일: 2026-04-03
상태: 참고 번역/정리

## 개요
Karpathy는 최근 다양한 연구 주제에 대해 LLM으로 개인 지식 베이스를 구축하는 방식을 유용하게 사용하고 있다고 설명한다. 이 과정에서 자신의 토큰 사용은 코드 조작보다 지식 조작에 더 많이 쓰이고 있으며, 그 지식은 주로 마크다운과 이미지 형태로 저장된다고 말한다.

## Data ingest
- 기사, 논문, 저장소, 데이터셋, 이미지 같은 원본 자료를 `raw/` 디렉토리에 넣는다.
- LLM이 이를 점진적으로 "컴파일"하여 위키를 만든다.
- 이 위키는 디렉토리 구조를 가진 `.md` 파일 모음이다.
- 위키에는 원본 요약, 백링크, 개념별 분류, 관련 문서 작성, 상호 연결이 포함된다.
- 웹 기사는 Obsidian Web Clipper로 `.md`로 변환하고, 관련 이미지는 로컬에 저장해 LLM이 함께 참조할 수 있게 한다.

## IDE
- Obsidian을 IDE의 프론트엔드처럼 사용한다.
- raw 데이터, 컴파일된 위키, 파생 시각화를 한 곳에서 볼 수 있다.
- 중요한 점은 위키의 대부분을 사람이 직접 관리하지 않고, LLM이 작성·유지한다는 것이다.
- Marp 같은 플러그인으로 슬라이드 같은 다른 표현도 볼 수 있다.

## Q&A
- 위키가 충분히 커지면 LLM 에이전트에게 복잡한 질문을 던질 수 있다.
- 에이전트는 위키를 바탕으로 답을 찾고 추가 조사를 수행한다.
- 복잡한 RAG를 쓰지 않아도, 인덱스와 요약이 잘 유지되면 비교적 작은 규모에서는 충분히 작동한다고 본다.

## Output
- 답변을 단순 텍스트 대신 마크다운 문서, Marp 슬라이드, matplotlib 이미지 등으로 출력할 수 있다.
- 그 결과물을 다시 위키에 filing 하여 다음 질의와 탐색에 활용한다.
- 즉 질의와 탐색이 곧 지식 베이스를 성장시키는 과정이 된다.

## Linting
- LLM으로 위키 건강 검사를 돌려 데이터 불일치, 누락, 흥미로운 연결, 새 문서 후보 등을 찾는다.
- 이를 통해 위키의 데이터 무결성과 품질을 점진적으로 높인다.

## Extra tools
- 위키 전용 간단 검색 엔진 같은 도구를 직접 만들 수 있다.
- 이를 웹 UI로 직접 쓰거나, CLI 도구로 LLM에 넘겨 더 큰 질의에 활용할 수 있다.

## Further explorations
- 저장소가 커질수록 합성 데이터 생성과 파인튜닝을 통해, LLM이 단순 컨텍스트 창이 아니라 가중치 수준에서 지식을 더 잘 알게 만드는 방향도 고려할 수 있다.

## 요약
원본 자료를 수집하고, LLM이 이를 `.md` 위키로 컴파일하며, 다양한 CLI와 질의를 통해 위키를 지속적으로 강화하고, 전체 과정을 Obsidian에서 열람하는 방식이다. 위키를 사람이 직접 쓰고 고치기보다 LLM이 관리한다는 점이 핵심이다.
