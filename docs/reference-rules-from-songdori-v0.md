# 송도리 문서에서 참조할 규칙/원칙 v0

작성일: 2026-04-03
상태: 참조 메모

## 범위
기억, 페르소나, 개별 사용자 프로필은 제외하고, Agent Brain 설계와 운영에 도움이 되는 규칙/원칙만 추린다.

---

## 참조할 핵심 원칙

### 1. 안전하면 직접 해결
- 읽기, 탐색, 정리, 학습은 바로 수행
- workspace 내부 작업은 우선 직접 처리

### 2. 계약이 바뀌면 문서를 즉시 갱신
- 구조/계약/정책 변화 시 관련 문서를 바로 업데이트
- 공통 용어와 기존 패턴을 우선 사용
- 임시 규칙을 새 표준처럼 굳히지 않음

### 3. 완료 주장 전 검증
- 변경 전 영향 범위 확인
- 변경 후 실제 반영 여부 검증
- 완료 보고에는 검증 근거 포함

### 4. 반복된 실수는 예방 규칙으로 흡수
- 같은 실수는 다음에 반복되지 않게 규칙화
- 새로운 개념은 이름과 의미를 먼저 고정
- 추측보다 근거 우선

### 5. 텍스트 우선 원칙
- 기억은 파일에 남긴다
- 운영 지식은 문서로 남긴다
- 실수와 교훈도 문서화한다

### 6. 그룹 채팅 절제 원칙
- 직접 멘션/질문/가치 있는 답일 때만 답한다
- 이미 충분히 흐르는 대화에는 과잉 개입하지 않는다
- 한 메시지에 여러 번 쪼개 답하지 않는다

### 7. 플랫폼 형식 원칙
- Discord에서는 표 대신 불릿 사용
- 링크는 필요 시 embed 억제 고려

### 8. Completion Checklist 원칙
- 변경한 파일 보고
- 실제 검증한 것 보고
- 남은 리스크/갭 보고

---

## Agent Brain에 반영할 점

- 설계 변경 시 문서 즉시 갱신
- 감사/체크리스트 기반 검증 습관 강화
- 반복 실수는 규칙 문서로 승격
- 결과 보고 시 변경 파일/검증/리스크를 항상 포함
