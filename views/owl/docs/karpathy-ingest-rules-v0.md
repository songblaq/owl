# Karpathy식 지식화 규칙 v0

작성일: 2026-04-03
상태: 운영 초안

## 1. 목적

이 문서는 owl 안에서 Karpathy식 방식으로 자료를 넣고 지식화하는 최소 규칙을 정의한다.

핵심은 다음과 같다.
- 원본을 파일로 보존한다.
- LLM이 원본을 읽고 마크다운 지식으로 컴파일한다.
- 결과물은 다시 지식 자산으로 축적한다.

---

## 2. 기본 흐름

```text
input -> inbox/raw 저장 -> source 정리 -> compiled 문서 생성 -> 필요 시 report/note/index 생성 -> 다시 filing
```

---

## 3. 자료 투입 규칙

### 3.1 무엇을 넣을 수 있나
- 대화 로그
- 웹 기사
- 논문
- GitHub 저장소 설명
- 링크 모음
- 메모
- 이미지 설명 자료

### 3.2 어디에 넣나
- 아직 정리 전이면 `~/owl-vault/inbox/`
- 원본 보관이면 `~/owl-vault/raw/`
- 정리 결과면 `~/owl-vault/compiled/`

### 3.3 원본 저장 원칙
- 가능한 원문을 보존한다.
- 날짜, 출처, 맥락을 남긴다.
- 원본은 나중에 다시 읽을 수 있어야 한다.

---

## 4. 컴파일 규칙

### 4.1 컴파일의 의미
컴파일은 단순 요약이 아니라 다음을 포함한다.
- 핵심 내용 추출
- 개념 분류
- 관련 항목 연결
- 다음 질문 제안
- 후속 문서 생성 기반 마련

### 4.2 최소 산출물
하나의 raw 자료가 들어오면 최소한 다음 중 하나를 만든다.
- summary 문서
- note 문서
- report 문서

### 4.3 문서 품질 기준
- 출처가 있어야 한다.
- 이 자료가 왜 중요한지 보여야 한다.
- 다음 작업 또는 관련 항목이 있어야 한다.

---

## 5. 파일명 규칙

### raw
- `YYYY-MM-DD-주제-raw.md`
- 예: `2026-04-03-owl-discord-thread-raw.md`

### compiled
- `YYYY-MM-DD-주제-summary.md`
- `YYYY-MM-DD-주제-note.md`
- `YYYY-MM-DD-주제-report.md`

---

## 6. 첫 적용 원칙

처음에는 복잡하게 하지 않는다.

1. 원본을 raw에 넣는다.
2. summary 하나를 만든다.
3. 필요하면 note 또는 report를 추가한다.
4. 나중에 index/관계 문서를 붙인다.

---

## 7. 검증과 보고 원칙

- 자료를 넣었다고 말하기 전에 실제 raw/compiled 생성 여부를 확인한다.
- 지식화 완료를 주장할 때는 생성된 파일 경로와 남은 갭을 함께 보고한다.
- 반복 실수가 생기면 ingest 규칙 자체를 수정한다.

## 8. 현재 운영 선언

owl은 이제 Karpathy식 지식화 작업을 시작할 수 있다.
앞으로 사용자가 자료를 주면, 우선 raw에 보관하고 compiled 문서를 생성하는 방식으로 처리한다.
